# Как реализовать использование Istio для дистрибутива выше 

## Для каких целей? 
- Контроль входящего и исходящего трафика
- Прозрачность этого трафика
- mTLS внутри кластера между сервисами
- Управление трафиком
- Дополнительные возможности, например ретраи и таймауты

## Какие сущности istio можно использовать? 
- ServiceEntry для описания внешних узлов, к которым можно подключаться через egress
- VirtualService для управления входящим трафиком
- DestinationRule для дополнительного роутинга трафика после VirtualService
- PeerAuthentication для настройки mTLS: не обязателен, обязателен или по желанию
- Gateway позволяет настраивать входящий и исходящий трафик

## Как лучше реализовать входной и выходной трафик?
- Все подключения из сети должны проходить через egress и быть разрешенными через настройки service mesh
- Входящий трафик так же должен проходить только через istio
- Если между разными кластерами, в базы,  планируется передаваться чувствительная информация, то обязательно нужно использовать mTLS
